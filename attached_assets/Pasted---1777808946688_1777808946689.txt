أريد تطوير تطبيق "المتحدث الصغير" ليصبح منصة تعليمية تفاعلية كاملة لتعليم الأطفال فن الإلقاء والخطابة، تجمع بين التعلم الذاتي والتدريب المباشر عبر Zoom.

المطلوب:
أنشئ نظامًا داخل التطبيق يحتوي على أدوات تعليمية تساعدني كمدربة على وضع المادة التعليمية، متابعة الطلاب، عقد حصص مباشرة، إرسال ملاحظات، وتقييم أداء الأطفال.

اللغة والواجهة:
- التطبيق كامل باللغة العربية.
- اتجاه RTL من اليمين إلى اليسار.
- واجهة سهلة للأطفال وأولياء الأمور والمدربة.
- تصميم مبهج واحترافي ومناسب للأطفال.
- يجب أن يعمل على الجوال والتابلت والكمبيوتر.

الأدوار داخل التطبيق:
1. المدربة / المشرفة
2. الطالب
3. ولي الأمر

أنشئ لوحة تحكم لكل دور.

أولًا: أدوات المدربة
أنشئ لوحة تحكم للمدربة تحتوي على:

1. إدارة الدورة
- إنشاء دورة جديدة.
- تقسيم الدورة إلى وحدات ودروس.
- إضافة عنوان الدرس.
- إضافة وصف قصير.
- رفع ملفات PDF أو صور أو فيديوهات.
- إضافة روابط خارجية.
- إضافة تمارين وتحديات.
- ترتيب الدروس بالسحب أو أزرار الترتيب.

2. ربط الدروس مع Zoom
- لكل درس مباشر، أريد خانة لإضافة رابط Zoom.
- إضافة تاريخ ووقت الحصة.
- زر واضح للطلاب بعنوان "انضمام إلى الحصة عبر Zoom".
- عرض الحصص القادمة في لوحة الطالب.
- إرسال تنبيه داخل التطبيق قبل موعد الحصة.
- بعد انتهاء الحصة، يتحول الدرس إلى حالة "تمت الحصة".
- أضف خانة لتسجيل رابط تسجيل الحصة إن توفر.

مهم:
في النسخة الأولى لا تحتاج إلى تكامل Zoom API حقيقي. يكفي أن تكون هناك خانات لإضافة رابط Zoom، وقت الحصة، رابط التسجيل، وزر دخول مباشر للرابط.

3. الرسائل بين المدربة والطلاب
- أنشئ نظام رسائل داخلي.
- المدربة تستطيع إرسال رسالة لطالب واحد أو لجميع طلاب الدورة.
- الطالب أو ولي الأمر يستطيع الرد على المدربة.
- الرسائل تظهر بشكل محادثة بسيطة.
- أضف حالة: مقروء / غير مقروء.
- أضف تنبيه للرسائل الجديدة.

4. التقييمات والملاحظات
- المدربة تستطيع تقييم كل طالب بعد كل درس أو تمرين.
- نموذج التقييم يحتوي على:
  - وضوح الصوت
  - الثقة بالنفس
  - ترتيب الأفكار
  - لغة الجسد
  - التواصل البصري
  - الالتزام بالتدريب
- التقييم يكون من 1 إلى 5 نجوم.
- خانة ملاحظات نصية خاصة من المدربة.
- خانة "ما يحتاج الطفل إلى تحسينه".
- خانة "نقطة قوة الطفل".
- عرض سجل تقييمات الطالب في ملفه.

5. متابعة تقدم الطلاب
- قائمة بجميع الطلاب.
- نسبة إنجاز كل طالب.
- عدد الدروس المكتملة.
- آخر نشاط قام به الطالب.
- آخر تقييم حصل عليه.
- فلترة الطلاب حسب المستوى أو التقدم.
- فتح ملف أي طالب لمراجعة أدائه.

ثانيًا: أدوات الطالب
أنشئ لوحة طالب تحتوي على:

1. مسار التعلم
- عرض الوحدات والدروس بشكل رحلة تعليمية.
- كل درس يحتوي على:
  - فيديو أو ملف أو نص قصير
  - تمرين
  - تحدي
  - زر "أنهيت الدرس"
  - عدد النجوم المكتسبة

2. دروس Zoom
- عرض الحصص المباشرة القادمة.
- زر "انضمام إلى Zoom".
- عرض وقت الحصة.
- عرض رابط تسجيل الحصة بعد إضافته من المدربة.
- حالة الحصة: قادمة، اليوم، انتهت.

3. التعلم الذاتي
أضف أدوات تساعد الطالب على التدرب وحده:
- تسجيل الصوت وتشغيله.
- تمرين قراءة جملة بصوت واضح.
- تمرين ترتيب الأفكار قبل الكلام.
- تمرين اختيار بداية قوية للحديث.
- تمرين لغة الجسد.
- تمرين التنفس قبل الإلقاء.
- تحدي تقديم خطاب قصير.
- حفظ محاولات الطالب داخل ملفه.

4. التقييم الذاتي
بعد كل تمرين، يظهر للطالب تقييم ذاتي بسيط:
- هل كان صوتي واضحًا؟
- هل تكلمت بثقة؟
- هل رتبت أفكاري؟
- هل استخدمت لغة جسدي؟
- هل أحتاج إلى إعادة المحاولة؟
الإجابة تكون بالنجوم أو الوجوه التعبيرية.

5. الإنجازات
- نظام نجوم.
- شارات إنجاز مثل:
  - صوت واضح
  - متحدث واثق
  - بداية قوية
  - قائد صغير
  - بطل التحدي
- شهادة نهاية الدورة قابلة للطباعة أو التنزيل.

ثالثًا: أدوات ولي الأمر
أنشئ لوحة ولي أمر بسيطة تحتوي على:
- تقدم الطفل.
- التقييمات والملاحظات من المدربة.
- الحصص القادمة عبر Zoom.
- الرسائل مع المدربة.
- إنجازات الطفل وشهادته.

رابعًا: أدوات بناء المادة التعليمية
أريد نظامًا يساعدني عندما أضع المادة داخل التطبيق:

1. منشئ الدروس
- خانة عنوان الدرس.
- خانة الهدف من الدرس.
- خانة المحتوى.
- خانة التمرين.
- خانة التحدي.
- خانة الواجب.
- خانة الملاحظات.
- اختيار نوع الدرس: ذاتي / مباشر عبر Zoom / واجب / تقييم.

2. منشئ التمارين
أنشئ أنواع تمارين جاهزة:
- سؤال اختيار من متعدد.
- ترتيب بطاقات.
- كتابة إجابة قصيرة.
- تسجيل صوتي.
- رفع ملف أو صورة.
- تمرين تقييم ذاتي.
- تحدي إلقاء قصير.

3. مكتبة المحتوى
- مكان لحفظ الملفات والدروس والتمارين.
- إمكانية إعادة استخدام تمرين في أكثر من درس.
- تصنيف المحتوى حسب الوحدة أو المهارة.

خامسًا: المتطلبات التقنية
استخدم:
- React + Vite
- واجهة RTL عربية
- localStorage في النسخة الأولى لحفظ البيانات
- Web Audio API للتسجيل الصوتي
- مكونات واضحة ومنظمة
- بيانات تجريبية جاهزة للطلاب والدروس والتقييمات
- تصميم responsive

لا أحتاج في البداية إلى نظام تسجيل دخول حقيقي أو قاعدة بيانات خارجية، لكن صمم الكود بطريقة تسمح بإضافة ذلك لاحقًا.

سادسًا: الصفحات المطلوبة
أنشئ الصفحات التالية:
1. الصفحة الرئيسية داخل التطبيق
2. لوحة المدربة
3. منشئ الدورات والدروس
4. إدارة روابط Zoom والحصص المباشرة
5. الرسائل
6. تقييم الطلاب
7. متابعة تقدم الطلاب
8. لوحة الطالب
9. صفحة الدرس
10. أدوات التعلم الذاتي
11. التسجيل الصوتي
12. الإنجازات والشهادات
13. لوحة ولي الأمر

مهم جدًا:
اجعل التطبيق عمليًا وليس مجرد شكل. أريد أزرارًا تعمل، بيانات تجريبية، إمكانية إضافة درس، إضافة رابط Zoom، إرسال رسالة، كتابة تقييم، وتسجيل صوت الطالب.

ابدأ ببناء نسخة MVP كاملة ومنظمة، ثم تأكد أن التطبيق يعمل بدون أخطاء.
